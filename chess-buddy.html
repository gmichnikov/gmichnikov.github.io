Chess Buddy Github

A terminal-based chess game that offers 1-player and 2-player mode

Chess Buddy is a basic Chess game written entirely in ruby. It allows the user(s) to play Human vs. Human or Human vs. Computer. Users can also watch a Computer vs. Computer game play out.

Chess Buddy has two primary foci. The first is extensive use of class inheritance to structure the code for the six different chess pieces. Rooks, Bishops, and Queens all inherit from a SlidingPiece class. Each of these pieces is able to move multiple spots in one of several directions, ultimately landing on an empty square or an opponent's piece. These shared abilities are captured in the parent class, while the differences (namely, diagonal vs. horizontal/vertical movement) is reserved for each piece's class. Similarly, Kings and Knights have a common SteppingPiece parent class, as each of these pieces can only move to 8 specififc squares on the board. The specific movements to get to those squares (e.g. up 2 over 1 vs.up 1 over 1) are in the child classes.

The other focus is a custom Computer Player AI. Chess Buddy categorizes all valid moves on several dimensions:

  1. In Check: Does the move put the opponent in check?
  2. Safe vs. At Risk: Does the move put given piece in a position where the opponent could immediately capture it on the next turn?
  3. Capture: Does the move capture a piece?
  4. Valuable Capture: Does the move capture a piece with value greater than or equal to the value of the capturing piece?

Chess Buddy's AI then chooses a move for the Computer Player in the following order of preference: 1. Safe and Valuable Capture 2. Safe and In Check 3. Valuable Capture 4. Safe and Capture 5. Safe 6. In Check 7. Capture 8. None of the above. In order to play Chess Buddy, users can download the code from Github and run "ruby chess.rb" in Terminal. More detailed instructions can be found on the Github page.
